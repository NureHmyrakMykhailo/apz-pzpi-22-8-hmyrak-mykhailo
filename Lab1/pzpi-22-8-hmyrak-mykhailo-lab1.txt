     1.2 Завдання
     
     
     Мета завдання – створити документ Vision & Scope для програмної системи для автоматизації роботи бібліотек. Цей документ має чітко окреслити бачення та межі проєкту, визначаючи його актуальність, цілі, ключові функціональні можливості, а також основні технічні та організаційні аспекти.
     Документ повинен служити спільною точкою орієнтації для всіх зацікавлених сторін — замовників, розробників, тестувальників, аналітиків та потенційних користувачів. Він допоможе уникнути розбіжностей у трактуванні вимог, встановити єдине розуміння завдань і забезпечити контроль обсягу робіт на всіх етапах розробки.
     
     
     1.3 Опис виконаної роботи
     
     
     У процесі розробки документа Vision & Scope для системи автоматизації бібліотек було детально визначено концептуальні основи та технічні параметри проекту. Документ охоплює всі критично важливі аспекти - від соціальної значущості до конкретних технічних рішень.
     Перший розділ документа присвячений аналізу сучасних проблем бібліотечної справи, зокрема неефективності ручних процесів обліку, складнощів у відстеженні місцезнаходження книг та відсутності систем моніторингу умов зберігання. Проведено глибокий аналіз існуючих рішень на ринку (таких як Koha та інші LMS), що дозволило виокремити унікальні переваги нашої системи, зокрема інтеграцію IoT-технологій та сучасний підхід до автоматизації. Також запропоновано інноваційні моделі монетизації, що поєднують підписну модель з додатковими платними послугами.
     Другий розділ чітко формулює основну мету проекту - створення комплексної системи для автоматизації ключових бібліотечних процесів, включаючи облік фондів, управління видачами та забезпечення оптимальних умов зберігання. Особливий акцент робиться на покращенні якості обслуговування читачів та зниженні операційних витрат бібліотек.
     Технічні аспекти системи детально розкриті у третьому розділі:
     * Серверна частина реалізована на C# з використанням .NET Core та Entity Framework для роботи з SQL Server, що забезпечує високу продуктивність та надійність
     * REST API розроблено з урахуванням сучасних стандартів безпеки та масштабованості
     * Веб-інтерфейс на ASP.NET Core MVC забезпечує зручне управління системою для адміністраторів
     * Мобільний додаток для Android дозволяє читачам швидко отримувати інформацію про наявність книг
     * IoT-компоненти на базі датчиків DHT22 та протоколу MQTT забезпечують моніторинг умов зберігання
     Четвертий розділ містить повний перелік зацікавлених сторін - від бібліотекарів і читачів до технічних фахівців та інвесторів. Детально описано план-графік розробки з чіткою прив'язкою до термінів та основних віх проекту. Визначено ключові технології, що будуть використовуватись на кожному етапі, з урахуванням необхідності інтеграції з існуючою інфраструктурою.

1 БІЗНЕС-ВИМОГИ
1.1 Передумови 
     
     Сучасні бібліотеки стикаються з серйозними труднощами через застосування застарілих методів управління. Основна проблема полягає в тому, що більшість установ досі використовують ручні системи обліку, що призводить до помилок у відстеженні книг, їх доступності та місцезнаходження. Це створює значні незручності як для працівників бібліотек, так і для читачів. Крім того, відсутність автоматизованих рішень ускладнює процеси видачі та повернення літератури, що часто викликає черги та незадоволеність користувачів.
     Важливою проблемою є також відсутність ефективного контролю за умовами зберігання бібліотечних фондів. Багато цінних видань псуються через невідповідну температуру або вологість, а бібліотеки не мають технічних можливостей для оперативного реагування на такі ситуації. Це особливо критично для архівних зібрань та рідкісних книг.
     Додатково ускладнює ситуацію те, що користувачі часто не мають зручного доступу до інформації про наявність книг або можливості дистанційного бронювання. Це знижує залученість читачів і робить бібліотеки менш привабливими у порівнянні з сучасними цифровими сервісами.

     1.2 Бізнес можливість
     
     Існуючі програмні рішення для бібліотек, такі як LMS або Koha, мають ряд суттєвих обмежень. Вони часто не передбачають інтеграції з сучасними технологіями, зокрема IoT-пристроями для моніторингу умов зберігання, а їх інтерфейси залишаються незручними для користувачів. Це створює значний простір для вдосконалення та розробки нових рішень.
     Наша система пропонує комплексний підхід до автоматизації бібліотечних процесів. Вона дозволяє не лише ефективно керувати фондами, але й забезпечує зручний сервіс для читачів через мобільний додаток та веб-інтерфейс. Інтеграція з IoT-пристроями дає можливість моніторити умови зберігання книг у реальному часі та запобігати їх пошкодженню.
     Ключовою перевагою нашої системи є її гнучкість та масштабованість. Вона може адаптуватися до потреб різних типів бібліотек - від невеликих публічних до великих академічних. Додатково система пропонує інструменти аналітики, які допомагають бібліотекам оптимізувати свою роботу та краще розуміти потреби користувачів.
     Перспективи розвитку системи включають інтеграцію з штучним інтелектом для прогнозування попиту на книги та надання персоналізованих рекомендацій. Це дозволить бібліотекам не лище покращити якість обслуговування, але й активізувати взаємодію з читачами. В майбутньому система може стати основою для створення єдиного мережевого простору бібліотек, що значно розширить їх можливості.
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     2 КОНЦЕПЦІЯ РІШЕННЯ
     2.1 Окреслення концепції
     
     Програмна система для автоматизації роботи бібліотек — це сучасне рішення, яке об’єднує ефективне управління бібліотечними фондами, автоматизацію рутинних процесів та IoT-моніторинг для забезпечення оптимальних умов зберігання книг.
     Ключові цілі:
     * Спростити облік книг: Заміна ручних процесів (журналів, карток) на цифрові.
     * Покращити досвід користувачів: Швидкий пошук, онлайн-бронювання, автоматичні нагадування.
     * Запобігти псуванню фондів: Контроль температури, вологості за допомогою датчиків.
     * Забезпечити звітність: Генерація документів для внутрішнього аудиту та перевірок.
     Цільові користувачі:
     * Бібліотекарі (додавання книг, управління видачами).
     * Читачі (пошук, бронювання, перегляд каталогу).
     * Адміністратори (налаштування системи, аналітика).
     * Технічний персонал (підтримка IoT-пристроїв).
     Унікальність рішення:
     * Інтеграція з IoT для проактивного реагування на зміни умов зберігання.
     * Модульна архітектура: Базовий функціонал + платні розширення (наприклад, AI-аналітика).
     * Відкриті API для інтеграції з іншими системами (наприклад, електронними каталогами університетів).
     
     
     2.2 Головна функціональність

     Для користувачів:
     * MF-1: Реєстрація/авторизація через email.
     * MF-2: Пошук книг за назвою, автором, жанром.
     * MF-3: Електронний каталог у реальному часі.
     * MF-4: Автоматизована видача/повернення книг.
     * MF-5: IoT-моніторинг умов зберігання.
     Для адміністраторів:
     * MF-6: Управління користувачами та ролями.
     * MF-7:Додавання/редагування книг, відстеження їх місцезнаходження.
     * MF-8: Налаштування правил (ліміти видач, резервування).
     * MF-9: Статистика активності (наприклад, популярність книг).
     У межах бізнес-логіки програмної системи реалізовано – при повернення книг система дозволяє оцінити середній час користування книгою. Це дозволяє зрозуміти, наскільки довго читачі утримують видані примірники, що корисно для аналітики попиту та оптимізації бібліотечного фонду.
     Середній час користування книгою обчислюється за формулою:
                                T_avg=  1/m ?_(j=1)^m??t_j^ret- t_j^out ?                                   (1)
     де T_avg  — середній час користування одним примірником, m — кількість завершених сесій видачі (тобто таких, де книгу вже повернуто), t_j^ret? — час повернення книги у j-й сесії, t_j^out? — час видачі книги у j-й сесії.
     Це дозволяє відслідковувати середню тривалість читання книг користувачами бібліотеки. У випадку значних відхилень від встановленого терміну користування система може автоматично генерувати попередження або рекомендації адміністрації.
     
     
     3 РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ
     	3.1 Рамки первинного випуску
     
     Перший реліз системи буде зосереджений на реалізації ключових функцій, необхідних для базової роботи бібліотеки:
     1. Автоматизований облік бібліотечних матеріалів:
     * Реєстрація нових книг з можливістю сканування ISBN
     * Каталогізація за основними параметрами (назва, автор, жанр, рік видання)
     * Відстеження статусу (наявність, видача, бронювання)
     2. Модуль управління доступом:
     * Розподіл прав за ролями (читач, бібліотекар, адміністратор)
     * Авторизація через email/соцмережі
     * Контроль дій користувачів
     3. Базовий моніторинг умов зберігання:
     * Підключення датчиків температури/вологості
     * Відображення показників у веб-інтерфейсі
     * Попередження про критичні відхилення
     Архітектура системи:
     Серверна частина:
     * Розроблена на C#
     * Використовує SQL Server для зберігання даних
     * Надає REST API для взаємодії з клієнтами
     * Включає модулі для обробки даних з IoT-пристроїв, генерації базових звітів, логування подій системи, автентифікації користувачів
     Клієнтські інтерфейси:
     1. Веб-додаток:
     * Адміністративна панель для управління користувачами та їх правами, бібліотечним фондом, системними налаштуваннями
     * Інтерфейс для бібліотекарів: Облік видач/повернень, перегляд статистики
     * Адаптивний дизайн для різних пристроїв
     2. Мобільний додаток:
     * Пошук книг у каталозі
     * Перегляд доступності
     * Особистий кабінет читача
     IoT-компоненти:
     * Використовують датчики DHT22 (температура/вологість)
     * Підключаються через Wi-Fi/мобільний інтернет
     * Передають дані по протоколу MQTT
     * Інтервал оновлення - 5 хвилин (налаштовується)
     Вимоги до системи:
     * Шифрування конфіденційної інформації
     * Масштабована архітектура
     * Підтримка паралельної роботи багатьох користувачів
     Система буде доступна через:
     * Веб-браузери 
     * Мобільний додаток 
     * API для інтеграції з іншими системами
     Перший реліз забезпечить всі необхідні функції для ефективного управління бібліотекою, з можливістю подальшого розширення функціоналу в наступних версіях.
     
     3.2 Рамки наступних випусків

     Після реалізації MVP-функціональності програмна система буде розширюватися за кількома ключовими напрямами. Одним із них стане інтеграція з іншими системами, зокрема реалізація відкритого API для підключення до університетських бібліотечних каталогів та зовнішніх електронних ресурсів, таких як Google Books. Це дозволить автоматично збагачувати базу книг додатковими метаданими, а також розширить можливості користувачів у пошуку й аналізі літератури.
     Система також отримає розширену підтримку звітності. Планується впровадження кастомізованих шаблонів для звітів, які можна буде адаптувати під конкретні потреби, наприклад, для внутрішнього аудиту. Крім того, дані з бібліотеки будуть візуалізовані за допомогою сучасних BI-інструментів, зокрема Power BI, що дозволить отримувати наочні графіки й аналітичні панелі в реальному часі.
     Ще одним напрямом розвитку є впровадження модуля AI-аналітики. Завдяки алгоритмам машинного навчання система зможе прогнозувати майбутню популярність книг на основі історичних даних видач. Також буде реалізовано рекомендаційний механізм, який персоналізовано пропонуватиме книги читачам, враховуючи їхні вподобання та попередні вибори.
     IoT-функціональність також розширюватиметься. Зокрема, буде додано підтримку розумних полиць з RFID-мітками, що дозволить у реальному часі відстежувати місцезнаходження кожної книги у бібліотеці. Система також зможе автоматично повідомляти про потенційне псування примірників — наприклад, при виявленні надмірної вологості на місці зберігання.
     У майбутніх релізах буде реалізовано багатомовний інтерфейс, який дозволить користувачам перемикатися між українською, англійською та іншими мовами. Це відкриє доступ до системи для ширшого кола користувачів, включно з іноземними студентами або бібліотеками за межами країни.
     Крім того, передбачається запровадження моделі монетизації через преміум-підписку. Вона надаватиме розширені функції для бібліотек, такі як додаткова аналітика, доступ до спеціальних модулів або пріоритетна технічна підтримка.
     

     3.3 Обмеження та винятки
     
     Система розроблена з акцентом на онлайн-роботу, тому повноцінний автономний режим не підтримується. Критичні операції, такі як списання книг або зміна системних налаштувань, вимагають обов'язкового підключення до центрального сервера. Це обмеження введено для забезпечення цілісності даних та запобігання конфліктам інформації при офлайн-роботі.
     Керування IoT-пристроями, зокрема налаштування датчиків температури та вологості, потребує участі технічного персоналу. Система не надає інструментів для самостійного конфігурування апаратного забезпечення, оскільки це може призвести до неправильної роботи датчиків і, як наслідок, недостовірного моніторингу умов зберігання.
     Ці обмеження введені свідомо для забезпечення стабільності, безпеки та простоти використання системи, зосередивши її функціонал на ключових задачах автоматизації бібліотечних процесів.

     4.3 Робоче середовище
     
     Серверна частина системи розроблена на сучасному технологічному стеку, що забезпечує високу продуктивність та надійність. В якості основної мови програмування обрано C# з використанням платформи .NET Core, що дозволяє створювати кросплатформенні рішення. Для побудови веб-додатку застосовується фреймворк ASP.NET Core, який забезпечує швидку розробку RESTful API. Доступ до даних реалізований через Entity Framework Core - потужний ORM-інструмент для роботи зреляційними базами даних.
     Система використовує SQL Server як основну базу даних для зберігання всієї критично важливої інформації: каталогу книг, даних користувачів, історії транзакцій та системних налаштувань. Для забезпечення безпеки реалізовано JWT-аутентифікацію з ролівою моделлю доступу, а всі конфіденційні дані шифруються за допомогою алгоритму AES-256.
     Клієнтська частина включає два основних інтерфейси взаємодії. Веб-додаток розроблений з використанням React.js та TypeScript, що забезпечує швидкий та інтерактивний користувацький досвід. Для побудови інтерфейсу застосовано бібліотеку Material-UI, яка надає сучасні та адаптивні компоненти. Система підтримує різні ролі користувачів, надаючи кожному відповідний набір функцій.
     Мобільний додаток для платформи Android розробляється в Android Studio. Особливістю додатка є підтримка обмеженого офлайн-режиму, що дозволяє користувачам отримувати доступ до ключових функцій навіть за відсутності інтернет-з'єднання.
     IoT-компоненти системи складаються з апаратної та програмної частин. Для моніторингу умов зберігання книг використовуються надійні датчики DHT22 (температура/вологість) та мікроконтролери ESP8266. Обмін даними між пристроями та сервером відбувається через протокол MQTT, що забезпечує легку масштабованість та надійність. Інтервал зчитування даних становить 5 хвилин і може бути налаштований адміністратором.
     Безпека системи реалізована за допомогою шифрування даних, регулярно створюються резервні копії бази даних для запобігання втраті інформації
